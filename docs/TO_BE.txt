TO-BE — Целевое состояние бизнес-процессов 
Отдел внедрения и поддержки информационной безопасности (ИБ)

1. Основная цель изменений
Повышение уровня информационной безопасности за счёт внедрения криптографического алгоритма RIPEMD-160 для контроля целостности данных и безопасной передачи служебной информации.

2. Новый порядок обработки данных
— Перед отправкой файлов или сообщений сотрудники используют разработанную программу-хешировщик.
— Программа генерирует контрольную сумму (хеш) RIPEMD-160.
— Отправляемые данные сопровождаются хешем для подтверждения целостности.
— Получатель сверяет пришедший файл с хешем.

3. Криптографическая защита
— Используется RIPEMD-160 для хеширования данных.
— Хеш передаётся по защищенному каналу (HTTPS).
— Файлы хранятся в защищённой сетевой директории.
— Проводится автоматическая проверка целостности при загрузке.

4. Уменьшение рисков
— Исключается передача конфиденциальных данных без криптографической защиты.
— Минимизируется человеческий фактор.
— Контроль целостности позволяет исключить подмену или повреждение данных.

5. Результат
После внедрения:
— все критические данные защищены криптографически,
— каналы передачи безопасны,
— бизнес-процесс формализован и поддаётся контролю,
— аудит ИБ выполняется проще и точнее.
